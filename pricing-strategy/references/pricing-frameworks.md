# Pricing Research Frameworks

## Value-Based Pricing Framework

```
Customer perceived value:           $1000
Your price:                         $500  ← capture value here
Next best alternative:              $300
Your cost to serve:                  $50
```

Key insight: Price between next best alternative and perceived value. Cost is a floor, not a basis.

---

## Value Metrics Reference

The value metric = what you charge for. Should scale with value customers receive.

| Metric | Best For | Example |
|--------|----------|---------|
| Per user/seat | Collaboration tools | Slack, Notion |
| Per usage | Variable consumption | AWS, Twilio |
| Per feature | Modular products | HubSpot add-ons |
| Per contact/record | CRM, email tools | Mailchimp |
| Per transaction | Payments | Stripe |
| Flat fee | Simple products | Basecamp |

## Choosing Your Value Metric
1. Identify how customers get value — what outcome do they care about?
2. Map usage to value — correlate usage patterns with retention/expansion
3. Test for alignment — "As customer uses more of [metric], do they get more value?"

## Freemium vs. Free Trial

### Freemium works when:
- Product has viral/network effects
- Free users provide value (content, data, referrals)
- Low marginal cost to serve free users
- Clear usage limits create natural upgrade triggers

### Free Trial works when:
- Product needs time to demonstrate value
- B2B with buying committees
- Higher price points
- Product is sticky once configured

### Credit card upfront trade-off:
- With card: 40-50% trial-to-paid conversion, lower volume
- Without card: 15-25% conversion, higher volume

### Reverse Trial (recommended):
Start with full premium access → downgrade to free after trial → user experiences premium first

## When to Raise Prices

Signs it's time:
- Competitors have raised prices
- Prospects don't flinch at price
- High conversion rates (>40%)
- Very low churn (<3% monthly)
- Added significant value since last pricing

### Price Increase Strategies
1. **Grandfather existing** — new price for new customers only
2. **Delayed increase** — announce 3-6 months out, drive annual conversions
3. **Increase tied to value** — add features to justify
4. **Plan restructure** — clean slate, map existing customers to nearest fit

---

## Price Testing Methods

1. Geographic testing — test higher prices in new markets
2. New customer only — raise for new, compare conversion rates
3. Sales team discretion — test higher quotes through sales
4. Feature-based testing — add premium tier at higher price

### What to Measure
- Conversion rate at each price point
- Average revenue per user (ARPU)
- Total revenue (conversion × price)
- Customer lifetime value
- Churn rate by price paid

## Enterprise Pricing

Add "Contact Sales" when: deals exceed $10k+ ARR, custom contracts needed, procurement involved.

### Table Stakes for Enterprise
- SSO/SAML
- Audit logs
- Admin controls
- Uptime SLA
- Security certifications

### Pricing Models
- Per-seat at scale: Volume discounts (e.g., $15/user → $10/user at 100+)
- Platform fee + usage: Base fee + usage above thresholds
- Value-based: Price tied to customer outcomes
